# PDF Report — Handoff для редизайна

Документ для передачи дизайнеру или другой модели, чтобы развивать PDF-шаблон.

Основной генератор — [backend/services/natal_pdf_html.py](backend/services/natal_pdf_html.py), использует HTML/CSS + Playwright. ReportLab оставлен аварийным fallback. Число страниц зависит от полного набора аспектов и объёма structured-отчёта.

---

## Что отдаётся в PDF

PDF — это «персональный отчёт по натальной карте». На входе у функции:
- `user_name` (строка, может быть пустой)
- `birth_date` (строка `"YYYY-MM-DD"`)
- `birth_time` (строка `"HH:MM"` или None)
- `birth_city` (строка)
- `sun_sign`, `moon_sign` (строки EN: `"libra"`, `"cancer"`, ...)
- `asc_sign` (строка EN или None)
- `planets` — dict `{planet_key → {sign, sign_degree, house, retrograde}}` для 10 планет (sun…pluto)
- `houses` — list из 12 объектов `{number, sign, degree}`
- `aspects` — list `[{p1, p2, aspect, orb}]` для каждого найденного аспекта между планетами
- `reading` — core-разделы из канонического structured payload
- `descriptions` — представление того же payload по каждой планете, дому и аспекту; отдельная LLM-генерация для PDF не выполняется

## Структура страниц

### 1. Обложка (страница 1)
- Крупное золотое «НАТАЛЬНАЯ КАРТА» (28 pt, по центру)
- Сабтайтл «Полный персональный отчёт» (14 pt, обычный)
- Имя пользователя (тусклый)
- Дата + время рождения, город
- По центру в одну строку золотом: `☉ Весы · ☽ Рак · AC Рыбы`
- Список «В отчёте» (6 буллетов):
  - Данные рождения и ключевые точки карты
  - Баланс стихий и общий энергетический профиль
  - Все планеты в знаках
  - Все 12 домов гороскопа
  - Все найденные аспекты между планетами
  - Подробные описания и персональная интерпретация
- Футер «Astro TMA · персональный астрологический отчёт»

### 2. Ключевые данные карты
- Заголовок золотом, сабтайтл «Краткая навигация по основным точкам натальной карты»
- Карточка-плашка с фоном `#0e0b20`, скруглённые углы, в ней rows:
  - Имя · значение
  - Дата рождения · значение
  - Время рождения · значение или «не указано»
  - Город рождения · значение
  - Солнце · знак (по-русски)
  - Луна · знак
  - Асцендент · знак или «не рассчитан»
- Ниже: подзаголовок «Баланс стихий»
- Список 4 стихий: Огонь / Земля / Воздух / Вода, для каждой — `N из 10 планет`

### 3. Планеты в знаках — таблица
Подзаголовок «Таблица показывает знаки, градусы и дома основных планет».
По 10 планет (sun, moon, mercury, venus, mars, jupiter, saturn, uranus, neptune, pluto), каждая в одной строке:
```
☉ Солнце      ♎ Весы 24°51'       Дом 7
☽ Луна        ♋ Рак  3°12'        Дом 4
☿ Меркурий    ♏ Скорпион 12°8' ℞   Дом 8
…
```

### 4. Планеты в знаках — описания
По каждой планете заголовок золотом (формат: `☉ Солнце в Весах`), под ним абзац интерпретации (4-6 предложений, ~80-120 слов).

### 5. Дома гороскопа — таблица
12 строк:
```
Дом 1       ♓ Рыбы          24°51'    (Асцендент)   ← золотом
Дом 2       ♉ Телец          5°12'
…
Дом 4       ♋ Рак           18°4'     (IC)          ← золотом
…
```
Дома 1/4/7/10 — оси карты, помечаются золотом и подписаны `Асцендент / IC / Десцендент / MC`.

### 6. Дома — описания
По каждому дому: заголовок «Дом N — Знак», абзац интерпретации (~100 слов).

### 7. Аспекты — список
Сгруппированы по типу (в этом порядке):
- ☌ Соединение
- △ Трин
- ⚹ Секстиль
- □ Квадрат
- ☍ Оппозиция
- ⚻ Квинконс

Внутри группы — строки вида:
```
☌ Соединение                        ← подзаголовок группы (золото)
   ☉ Солнце ☌ ☿ Меркурий            орб 1.4°
   ♀ Венера ☌ ♂ Марс                орб 3.2°
△ Трин
   ☽ Луна △ ♃ Юпитер               орб 0.8°
…
```

### 8. Аспекты — описания
По каждому аспекту: заголовок «Солнце ☌ Меркурий — Соединение», абзац (~100 слов).

### 9. Персональная интерпретация
Core-часть того же канонического отчёта: основа личности, главные темы,
лунные узлы при наличии данных, заключительный синтез и практические
ориентиры. Фактические строки формирует backend; LLM-текст их не повторяет.

### 10. Финал
«Спасибо» крупно золотом по центру + «Этот отчёт создан для вашего понимания личного космического рисунка.» приглушённым.

## Что хочется получить (что просить у Claude)

1. **Полная типографическая иерархия** — H1/H2/H3/Body/Caption, межстрочные интервалы, ширина колонки.
2. **Сетка обложки** — крупный mandala/SVG-фон, расположение элементов. Сейчас всё центрировано flat'ом.
3. **Карточная схема** для секций «Планеты», «Дома», «Аспекты» — не плоские строки, а карточки с иерархией: глиф крупно, рядом знак+градус+дом, ниже короткое summary.
4. **Иконография** — astrological glyphs стилизованные, не Unicode-шрифтовые. (Можно использовать SVG из `frontend/public/zodiac-glyphs/` если редизайн будет через HTML→PDF.)
5. **Цветовая разметка** — текущая палитра (тёмный фон + золото). Можно слегка расширить (например для аспектов: trine/sextile — мягкий зелёный, square/opposition — тёплый красный, conjunction — золото).
6. **Графика круга карты** — SVG-натальной карты на одной из страниц (есть код в [frontend/src/components/NatalChart/](frontend/src/components/NatalChart/)).
7. **Лента содержания + номера страниц**.
8. **Финальная страница** — что-то получше «Спасибо».

## Технические ограничения

- Это PDF, генерируется на бэкенде Python.
- Сейчас инструмент — **ReportLab** (low-level Canvas API: `c.drawString`, `c.rect`, ручной wrap-paragraph).
- Альтернатива для редизайна: **WeasyPrint** (HTML+CSS → PDF) или **Playwright** (рендерит web-страницу в PDF) — даст полную свободу с CSS, web-fonts, SVG, gradients.
- Если переход на HTML/CSS подход — это **переписывание** генератора. Скажи если этот путь интересен — могу прикинуть migration cost.
